# 演出配置指南 —— 技能与表演联动说明

本文档以「小薇日记」为例，逐步说明如何创建知识库、配置技能、绑定演出音效。当前故事包共 57 个技能、26 个阶段（含 3 个分支点），演出条目覆盖角色语音、技能音效、阶段切换等多个维度。

> **v2 架构提示**：故事包现已支持 **Module-Flow 分离架构**（可选）。舞台（stage）被拆分为独立的 `modules`（故事模块）和 `flow`（流程控制），但演出配置完全兼容——无论 v1 还是 v2，`skills.json`、`knowledge/documents.json`、`manifest.json` 的联动关系不变。模块的 `guidance` 字段应指引 LLM 在何时使用哪些技能/触发词，从而自然地触发演出。详见 [AI 故事配置指南](./AI故事配置指南.md) 第八章。

---

## 一、两种触发模式对比

| | knowledgeUse（知识库命中） | skillUse（技能使用） |
|---|---|---|
| **触发依据** | 消息文本内容 | LLM 的 action JSON |
| **关键字段** | `knowledgeTitle` / `keywords` | `skillId` |
| **需要 skills.json** | 不需要 | 必须有 |
| **匹配方式** | 文本关键词包含匹配 | ID 精确相等 |
| **适用场景** | 任何反应/状态/台词 | 有数值结算的技能 |

## 二、例一：knowledgeUse —— 小薇「傲娇激烈反抗」

### 1. 知识库文档（粘贴到角色知识库编辑器）

```markdown
## 小薇调教反应知识库技能卡

使用规则：当小薇触发反应时，必须把反应名称写成 **粗体**。

## 傲娇激烈反抗                   ← ① 章节标题
- 类型：反抗反应
- 表演：傲娇激烈反抗             ← ② 关联到演出的桥梁字段
- 触发词：激烈反抗、傲娇反抗     ← ③ LLM 被要求输出的关键词
- 效果：短暂降低调教效果
- 发动时机：被捆绑固定时优先触发
- 描述：小薇会激烈挣扎、扭动身体并大声抗议
- 台词："放开我！你们在干什么！不要啊！"
- 输出要求：必须出现 **傲娇激烈反抗** ← ④ 强制 LLM 输出粗体
```

**每一行的作用：**

| 行 | 作用 |
|----|------|
| `## 章节标题` | 让 LLM 知道这是一条可触发的反应规则 |
| `表演：xxx` | **桥梁字段**——填的值与演出配置页的「知识库标题」对应 |
| `触发词：A、B` | LLM 会在回复中输出这些词（按要求用粗体包裹） |
| `输出要求` | 约束 LLM 的格式，确保触发词能被系统捕获 |
| 其余行（效果/时机/描述/台词） | 给 LLM 提供上下文，让它知道何时触发 |

### 2. 演出配置页填写

| 字段 | 值 | 说明 |
|------|-----|------|
| 名称 | 傲娇激烈反抗 | 显示名称 |
| 触发器 | 知识库命中 | 选择 `knowledgeUse` |
| 角色 | 小薇 (xiao_wei) | 限定只有小薇的消息才触发 |
| 知识库标题 | 傲娇激烈反抗 | **必须与知识库「表演：」值一致** |
| 触发词 | 激烈反抗、傲娇反抗 | **必须与知识库「触发词：」值一致** |
| 渲染器 | 声音 | 上传 mp3 |
| 播放频次 | 每次都播 | |

### 3. 保存后生成的 manifest.json 条目

```jsonc
"xiao_wei_fierce_resist": {
  "name": "傲娇激烈反抗",
  "renderer": "audio",
  "trigger": {
    // ↓ 来源：演出配置页「触发器」下拉
    "type": "knowledgeUse",
    // ↓ 来源：演出配置页「角色」下拉
    "characterId": "xiao_wei",
    // ↓ 来源：演出配置页「知识库标题」输入框
    "knowledgeTitle": "傲娇激烈反抗",
    // ↓ 来源：演出配置页「触发词」输入框
    "keywords": ["激烈反抗", "傲娇反抗", "不要", "放开"],
    // ↓ 来源：UI 固定为 true（可在 manifest.json 手动改）
    "matchBoldOnly": false
  },
  "audio": { "main": "assets/performances/xiao_wei_voice/audio/...mp3" }
}
```

### 4. 运行时链路

```
LLM 读知识库「傲娇激烈反抗」章节
  → LLM 判断时机合适，按「输出要求」生成回复：
    「小薇拼命扭动身体，**激烈反抗**着大喊："不要啊！"」
  → 前端 Runtime 收到消息，提取粗体文本 → "激烈反抗"
  → "激烈反抗" 归一化后匹配到 trigger.keywords 中的「激烈反抗」
  → 播放 xiaowei_short_reaction.mp3
```

## 三、例二：skillUse —— 小薇「本能反抗」

### 1. 角色配置页创建技能

| 字段 | 值 |
|------|-----|
| ID | `skill_resist` ← **全局唯一标识，所有联动靠这个** |
| 名称 | 本能反抗 |
| 消耗 | 15 MP |
| 伤害 | 5~10 |
| 效果 | 短暂降低当前调教效果 |
| 描述 | 小薇挣扎扭动身体 |
| 示例台词 | 不要！放开我！ |

保存后写入 `skills.json`：

```jsonc
{
  "id": "skill_resist",         // ← 这个 ID 连接一切
  "name": "本能反抗",
  "ownerId": "xiao_wei",        // 只有小薇能用
  "cost": { "mp": 15 },
  "damage": { "min": 5, "max": 10 },
  "effect": "短暂降低调教效果"
}
```

### 2. 知识库中写入技能用法

```markdown
## 本能反抗                       ← LLM 读到的技能说明
- 类型：反抗技能
- 效果：短暂降低调教效果
- 发动时机：每次调教刚开始，或被压制得喘不过气时使用
- 描述：小薇挣扎扭动身体，试图挣脱束缚
- 台词："不要！放开我！"
- 输出要求：回复中使用 **本能反抗**，
  并在 JSON action 中指定 type="skill", skillId="skill_resist"
                           ↑ 让 LLM 使用正确的技能 ID
```

### 3. 演出配置页新建演出

| 字段 | 值 |
|------|-----|
| 名称 | 本能反抗音效 |
| 触发器 | 技能使用 |
| 关联技能 | 下拉选择「本能反抗 (skill_resist)」← **与 skills.json 的 id 完全一致** |

生成的 manifest 配置：

```jsonc
"trigger": {
  "type": "skillUse",          // 触发类型
  "skillId": "skill_resist"    // 与 skills.json 的 id 完全一致
}
```

### 4. 运行时链路

```
LLM 判断时机合适 → 生成 JSON action：
  { "action": { "type": "skill", "skillId": "skill_resist" } }
  → 服务端 turnProcessor 验证 skill_resist 存在 → 结算伤害/消耗
  → turnProcessor 把 "skill_resist" 写入 message.usedSkills 数组
  → 前端 Runtime 遍历所有 skillUse 类型的演出
  → trigger.skillId="skill_resist" ∈ message.usedSkills → 命中 → 播放音效
```

## 四、新建反应/技能并配演出的完整流程

1. **确定类型** → 文字触发选 `knowledgeUse`，需要数值结算选 `skillUse`
2. **写知识库** → 按上面的模板写章节，`表演：xxx` 和 `触发词：xxx` 不能漏
3. **建技能**（如果是 skillUse） → 角色配置页创建，记下 ID
4. **配演出** → 演出配置页新建，选对应触发器，填匹配字段
5. **上传资源** → 拖拽 mp3/png，确认路径正确
6. **加 Prompt 规则** → 确保 LLM 知道要输出粗体或 JSON action
7. **测试** → 开 DevTools Console，发一句话看有没有触发音频加载

## 五、演出实战案例 —— 小薇 ⑪⑫ 多人桥段

⑪⑫ 是故事的高潮桥段，涉及三个角色（大姐、冷老师、小薇）同时协力，技能和表演配置较为复杂。

### 5.1 角色与技能分工

| 角色 | 阶段⑪ 技能 | 阶段⑫ 技能 | 定位 |
|------|-----------|-----------|------|
| 大姐 (da_jie) | 前后同步夹击术 | 临界抽撤术 | 主导调教 |
| 冷老师 (leng_ku_jiao_shi) | 前后同步夹击术（协同） | 临界抽撤术（协同） | 协同压制 |
| 小薇 (xiao_wei) | （无主动技能） | （无主动技能） | 被动反应 |

### 5.2 小薇被动反应表演（knowledgeUse）

小薇在此桥段的表演全部通过 reactive knowledgeUse 触发，由大姐/冷老师的技能知识库中的 `小薇反应：` 行联动输出：

| 表演 key | 知识库标题 | 触发词 | 场景 |
|----------|-----------|--------|------|
| `xiao_wei_gag_muffled` | 口塞闷哼 | 口塞闷哼、呜咽闷哼 | 口塞堵住嘴巴时，努力发出声音 |
| `xiao_wei_dual_attack_cry` | 夹击急促娇啼 | 夹击急促娇啼、前后同时 | 大姐和冷老师同时发动前后夹击 |
| `xiao_wei_gag_crying` | 口塞含混哭腔 | 口塞含混哭腔、含混不清 | 被压制到极限时的崩溃哭声 |
| `xiao_wei_vacuum_breakdown` | 刺激真空爆哭 | 刺激真空爆哭、崩溃大哭 | 三人协力刺激真空，彻底崩溃 |
| `xiao_wei_yield_fragments` | 屈服碎语乞求 | 屈服碎语乞求、屈服乞求 | 最终屈服，断断续续祈求停止 |
| `xiao_wei_aftermath_tremble` | 事后无力低泣 | 事后无力低泣、低泣 | 事后瘫软，只能小声抽泣 |

### 5.3 知识库联动写法

在大姐的知识库中，每个技能条目必须有 `小薇反应：` 行：

```markdown
## 前后同步夹击术
- 类型：调教技
- 表演：前后同步夹击术
- 触发词：前后同步夹击、前后同时进攻
- 效果：大姐与冷老师同时攻击小薇的前后敏感点
- 发动时机：进入⑪阶段后，与冷老师配合使用
- 描述：大姐在前方进攻，冷老师在后方配合
- 台词："冷老师，我们一起来。"
- 小薇反应：被前后夹击时出现**夹击急促娇啼**，身体剧烈颤抖。  ← 关键行
- 输出要求：必须出现 **前后同步夹击术**。
```

### 5.4 manifest.json 配置示例

```jsonc
"xiao_wei_dual_attack_cry": {
  "name": "夹击急促娇啼",
  "renderer": "audio",
  "durationMs": 3200,
  "trigger": {
    "type": "knowledgeUse",
    "characterId": "xiao_wei",
    "knowledgeTitle": "夹击急促娇啼",
    "keywords": ["夹击急促娇啼", "前后同时"],
    "matchBoldOnly": true
  },
  "playOnce": "never",
  "audio": {
    "main": "assets/performances/wuxia_sfx/audio/xiaowei_dual_cry.mp3"
  }
}
```

### 5.5 运行时链路（多角色联动）

```
大姐 LLM 读知识库「前后同步夹击术」章节
  → 看到「小薇反应：被前后夹击时出现**夹击急促娇啼**」
  → LLM 生成大姐的叙述，同时输出：
    「小薇被前后夹击，发出**夹击急促娇啼**，身体剧烈颤抖。」
  → 前端 Runtime 收到消息，提取粗体 → "夹击急促娇啼"
  → 匹配到 xiao_wei_dual_attack_cry 表演 → 播放音效
```

**要点**：小薇自己没有主动技能，表演全靠其他角色的知识库中的 `小薇反应：` 行来触发。

## 六、skillUse 实战案例 —— 前后同步夹击术

### 6.1 skills.json 定义

```jsonc
{
  "id": "skill_dual_front_rear",        // ← 全局唯一 ID
  "name": "前后同步夹击术",
  "ownerId": "da_jie",                  // 大姐专属
  "cost": { "mp": 25 },
  "damage": { "min": 18, "max": 28 },
  "effect": "与冷老师同步攻击，伤害+10，小薇大幅降低抵抗",
  "description": "大姐在前方攻击，冷老师在后方同步夹击，形成前后合围之势",
  "sampleLine": "冷老师，前后一起！"
}
```

### 6.2 演出配置（skillUse + knowledgeUse 双触发）

一个技能可同时配两条演出：
- **skillUse 演出** → 播放技能音效（如掌风、碰撞声）
- **knowledgeUse 演出** → 播放小薇反应语音

```jsonc
// 技能音效（skillUse）
"da_jie_dual_attack_skill": {
  "name": "前后夹击技能音效",
  "renderer": "audio",
  "trigger": {
    "type": "skillUse",
    "characterId": "da_jie",
    "skillId": "skill_dual_front_rear"
  },
  "audio": { "main": "assets/performances/wuxia_sfx/audio/dual_attack.mp3" }
}
```

## 七、常见问题

**Q: 技能不需要内力/伤害怎么办？**
A: 消耗和伤害都设为 0 即可。无战斗场景照样可以定义技能来触发演出。

**Q: LLM 总是不写粗体怎么办？**
A: 在提示词规则里加一条：「请将关键动作或状态用 **粗体** 标出」。

**Q: 怎么知道演出有没有触发？**
A: 打开浏览器 DevTools → Console，观察音频加载日志。

**Q: matchBoldOnly 是 true 还是 false？**
A: `true` = 只匹配消息中的粗体文字，精确度高。`false` = 全文匹配，更宽松但可能误触发。UI 目前固定为 `true`，如需 `false` 请手动编辑 manifest.json。
