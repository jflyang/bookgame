# 故事导出为小说 — 技术与设计方案

## 需求概述

选择一个故事包，根据其流程结构，用 AI 逐章节生成一份完整的小说文档（Markdown）。

核心特性：
- 可设置每章字数
- 按流程图顺序分批生成
- 分支路线由用户选择（不是全部生成）
- 循环事件只发生一次（或用户跳过）
- 实时进度展示
- 最终输出一份 `.md` 文件

---

## 用户交互流程

```
选择故事包
    ↓
[配置页] 设置参数：
  - 每章字数（默认 1500）
  - 写作风格（如：第三人称、文学性强、对话多）
  - 选择分支路线
  - 选择是否包含循环事件
  - 选择结局（如有多结局）
    ↓
[预览路线] 显示最终要生成的章节列表（线性化后的路径）
  - 用户可以跳过某些章节
  - 用户可以调整顺序
    ↓
点击「开始生成」
    ↓
[生成页] 逐章生成，实时显示进度
  - 每章生成完立即显示预览
  - 可以暂停/继续
    ↓
[完成] 下载 .md 文件
```

## 数据流设计

### Step 1: 路线解析

输入：
```json
{
  "packagePath": "/path/to/story",
  "branchChoices": {
    "node_choice_branch": 0  // 选择第 0 个分支（仁道正法）
  },
  "skipStages": ["stage_z2KkWy3V8B"],  // 跳过日常修炼
  "loopOnce": true  // 循环事件只执行一次
}
```

输出：
```json
{
  "chapters": [
    { "stageId": "stage_001", "title": "① 师门血债", "type": "training", "skipped": false },
    { "stageId": "stage_002", "title": "② 山路初遇", "type": "training", "skipped": false },
    ...
    { "stageId": "stage_007", "title": "⑦ 抉择", "type": "choice", "chosenBranch": "仁道正法" },
    { "stageId": "stage_008a", "title": "⑧A 逍遥正法", "type": "serving", "skipped": false },
    ...
  ],
  "totalChapters": 18,
  "estimatedWords": 27000
}
```

**路线解析算法：**
1. 从 `flow.json` 的 `node_start` 开始
2. 沿 edges 遍历，遇到 choice 节点时根据 `branchChoices` 选择分支
3. 遇到 loop 节点时：如果 `loopOnce=true`，只走一次循环体
4. 遇到 `skipStages` 中的节点，标记为 skipped
5. 到达 `node_end` 结束
6. 输出线性化的章节列表

### Step 2: 逐章生成

输入（每章）：
```json
{
  "packagePath": "/path/to/story",
  "stageId": "stage_001",
  "chapterIndex": 0,
  "totalChapters": 18,
  "targetWords": 1500,
  "style": "第三人称，文学性强，注重氛围描写",
  "previousSummary": "（上一章的 200 字摘要，提供连贯性）",
  "storyContext": {
    "title": "虚竹除害星宿老怪",
    "premise": "...",
    "characters": [...]
  }
}
```

输出：
```json
{
  "content": "# 第一章 师门血债\n\n少林寺藏经阁内，烛火摇曳...",
  "wordCount": 1523,
  "summary": "虚竹在藏经阁发现丁春秋欺师灭祖的证据，决定下山清理门户。乔峰和段誉表示同行。"
}
```

### Step 3: 组装 MD

将所有章节按顺序拼接：


```
你是一位专业的小说作家。根据故事大纲和当前章节的剧情指引，写出一章完整的小说内容。

要求：
1. 字数控制在 {targetWords} 字左右（允许 ±10% 浮动）
2. 写作风格：{style}
3. 必须包含 guidance 中描述的关键情节和对话
4. 保持与上一章的连贯性（参考 previousSummary）
5. 章节开头不需要重复标题
6. 对话用「」包裹
7. 动作描写用细腻的文学语言
8. 每章结尾留一个小悬念或过渡，引向下一章

输出格式：
- 直接输出小说正文（纯文本，不要 markdown 标记）
- 最后一行用 --- 分隔，写一个 200 字以内的章节摘要（供下一章参考）
```

## 路线解析算法（核心）

```typescript
function resolveLinearPath(
  nodes: FlowNode[],
  edges: FlowEdge[],
  branchChoices: Record<string, number>,  // nodeId → branchIndex
  skipStages: string[],                   // stageIds to skip
  loopOnce: boolean
): ChapterInfo[] {
  const chapters: ChapterInfo[] = [];
  const visited = new Set<string>();
  
  // Build adjacency map
  const outEdges = new Map<string, FlowEdge[]>();
  for (const edge of edges) {
    if (!outEdges.has(edge.source)) outEdges.set(edge.source, []);
    outEdges.get(edge.source)!.push(edge);
  }
  
  // BFS/DFS from start
  let current = nodes.find(n => n.type === "start")?.id;
  
  while (current) {
    if (visited.has(current)) break;  // Prevent infinite loops
    visited.add(current);
    
    const node = nodes.find(n => n.id === current);
    if (!node) break;
    
    if (node.type === "end") break;
    
    if (node.type === "module") {
      const stageId = node.data.moduleData?.sourceStage || node.data.moduleRef?.replace("mod_", "");
      const skipped = skipStages.includes(stageId);
      chapters.push({
        stageId,
        title: node.data.label,
        type: node.data.colorKey || "training",
        skipped,
        nodeId: current,
      });
    }
    
    if (node.type === "choice") {
      // User chose a branch
      const chosenIdx = branchChoices[current] ?? 0;
      const branchEdges = outEdges.get(current)?.filter(e => e.sourceHandle?.startsWith("branch_")) || [];
      const chosen = branchEdges.find(e => e.sourceHandle === `branch_${chosenIdx}`);
      current = chosen?.target || branchEdges[0]?.target;
      continue;
    }
    
    if (node.type === "loop") {
      // Enter loop body once
      const bodyEdge = outEdges.get(current)?.find(e => e.sourceHandle === "loop_body");
      const exitEdge = outEdges.get(current)?.find(e => e.sourceHandle === "loop_exit");
      
      if (loopOnce && bodyEdge) {
        // Walk the loop body once, then take exit
        // ... traverse loop body nodes until they point back to this loop node
        // Then continue from exitEdge.target
      }
      
      current = exitEdge?.target;  // Skip loop entirely if loopOnce and already visited
      continue;
    }
    
    // Default: follow first outgoing edge
    const nextEdge = outEdges.get(current)?.[0];
    current = nextEdge?.target;
  }
  
  return chapters;
}
```
