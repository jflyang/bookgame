# 故事包格式改造指南（旧 → 新）

> 基于虚竹除害星宿老怪 → 小薇的身体格式的实战改造，提炼为通用指南。

## 适用场景

将旧格式故事包（`linearPhases` + 内嵌 modules + `personaPrompt` 角色人设）改造为 React Flow 图模型 + 模块化行为定义的新格式。

---

## 一、新旧格式完整字段对照表

### 文件级差异

| # | 文件 | 旧格式 | 新格式 | 改造方式 |
|---|------|--------|--------|---------|
| 1 | `package.json` | `id: "故事名"` | `id: "story_xxx_XXXXXXXX"` | 加前缀 `story_` + 随机 ID |
| 2 | `manifest.json` | 含 rich performances | `{ title, description, version, performances: {} }` | 精简顶层，保留 performances |
| 3 | `scenario.json` | `stageType` 全 `"training"`，`directive` 全空，无 `isChoicePoint` | stageType 多样，directive 详细，抉择点有标记 | 按叙事阶段分配类型，补 directive |
| 4 | `flow.json` | `{ linearPhases, modules, finaleSequence }` | `{ nodes: [], edges: [] }` | 完全重写 |
| 5 | `modules.json` | ❌ 不存在（嵌在 flow.json） | ✅ 独立数组 | 从 flow.json 提取 `modules` 数组 |
| 6 | `characters.json` | 含 `personaPrompt`(长文本)、`rules:[]`、`voiceId` 不全 | 无 personaPrompt/rules，`voiceId` 全 | 删多余字段，补 voiceId |
| 7 | `actions.json` | ❌ | ✅ 角色行动定义 | 从 personaPrompt + knowledge 提取 |
| 8 | `reactions.json` | ❌ | ✅ 角色反应定义 | 从角色设定和剧情提取 |
| 9 | `skills.json` | ❌ | ✅ 技能定义 | 从 knowledge.json 技能卡提取 |
| 10 | `ui-config.json` | ❌ | ✅ `{}` | 创建空对象 |
| 11 | `knowledge.json` | 保持 | 保持 | 字段对齐（内容不动） |
| 12 | `rules.json` | 保持 | 保持 | 字段对齐 |
| 13 | `setting.md` | 保持 | 保持 | 不动 |
| 14 | `media/` | 保持 | 保持 | 不动 |
| 15 | `saves/` | 旧存档 | 可保留或删除 | -- |

## 二、stageType 分配规则

根据叙事阶段性质分配 stageType（6 种）：

| stageType | 含义 | 何时使用 |
|-----------|------|---------|
| `training` | 导入/教学 | 故事开局、角色登场、背景介绍、局势铺垫 |
| `choice` | 抉择点 | 玩家做出分支选择的关键节点 |
| `serving` | 正派行动/主角优势 | 主角反击、揭露真相、救死扶伤、以德服人、温暖收束 |
| `punishment` | 反派反扑/弱势危机 | 反派突袭、暗算得手、毒素发作、狗急跳墙、众叛亲离 |
| `daily` | 日常/循环 | 日常修炼、反复可执行的活动 |
| `event` | 高潮/特殊事件 | 大招释放（如生死符）、终极对决、反派崩溃 |

**分配原则：**
- `serving` 和 `punishment` 交替出现，形成叙事起伏（优势→反扑→再优势→再反扑）
- `training` 仅在开头使用（前 4-5 个阶段）
- `event` 用于高潮节点（每条约 2-3 个）
- `daily` 标记循环阶段（`reusable: true`）
- `choice` 有且仅有一个抉择点

## 三、nodes/edges 布局设计

### 布局原则
- 水平轴 = 剧情时间推进（左→右）
- 垂直轴 = 分支/并行路线（A 路径行 2, B 路径行 3, C 路径行 4）
- 节点间距：`COL_WIDTH = 280`, `ROW_HEIGHT = 160`
- START 放在最左侧上方，END 放在最右侧中间

### edge 类型
| 类型 | 何时使用 | edge 结构 |
|------|---------|----------|
| `plain_edge` | 普通线性过渡 A→B | `{ source: "node_A", target: "node_B" }` |
| `choice_edge` | 抉择点引出分支线 | `{ source: "node_choice", target: "node_branch", sourceHandle: "branch_0", label: "分支名" }` |
| `labeled_edge` | 需要标注的过渡（汇聚/Daily循环/终幕） | `{ source, target, label: "标注" }` |

### 循环阶段处理
```
⑯ 星宿解散 ──→ ⑰ 日常修炼 ──→ ⑯ (修炼完毕，循环)
                         └──→ ⑱ 归途同行 (踏上归途，退出循环)
```

## 五、改造其他故事包的步骤

### 快速三步

**Step 1: 备份**
```bash
cp -r apps/data/task-packages/<包名> apps/data/task-packages/<包名>_backup
```

**Step 2: 修改自动化脚本**
- 复制 `scripts/transform_xuzhu.cjs` → `scripts/transform_<包名>.cjs`
- 修改脚本中的以下变量：
  - `SRC` 路径指向目标故事包
  - `ACTIONS_BY_CHAR` — 根据角色重新定义行动
  - `REACTIONS` — 根据剧情重写反应
  - `STAGE_TYPE_MAP` — 根据叙事重新分配 stageType
  - `DIRECTIVES` — 为每个 stage 编写 directive
  - `LAYOUT` — 为每个 module 指定布局位置 (col, row)
  - `preBranchIds` / `pathA` / `pathB` / `pathC` — 调整分支结构
  - `DEFAULT_VOICE_IDS` — 更新角色 ID 和 voice ID 映射

**Step 3: 运行**
```bash
node scripts/transform_<包名>.cjs
```

### 必须手动调整的部分

以下内容无法自动生成，需要人工编写：

| 内容 | 说明 |
|------|------|
| `actions.json` 的 description | 根据角色人设编写行动描述 |
| `reactions.json` 的 trigger/description | 根据剧情编写触发条件和反应文字 |
| `skills.json` 的 effect/sampleLine | 技能效果分类 + 示例台词 |
| `scenario.json` 的 directive | 每阶段给 AI 的强制指令 |
| `flow.json` 的 LAYOUT 坐标 | 节点在画布上的可视化位置 |
| `flow.json` 的 edge 连接关系 | 分支/汇聚/循环的正确拓扑 |
